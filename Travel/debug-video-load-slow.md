# Debug Session

- Session ID: video-load-slow
- Started At: 2026-04-04
- Scope: 任务列表和预览区视频加载慢的原因排查

## Symptoms

- 任务列表里的视频加载慢
- 预览区视频加载慢

## Hypotheses

1. 页面在列表区域一次性渲染了多个 `<video>`，浏览器并发拉取媒体资源，导致首屏预览变慢。
2. 任务列表和预览区使用的都是完整 mp4 原片，没有封面图或更轻的预览资源，导致每次显示都要拉完整媒体头信息甚至更多字节。
3. 当前 `<video>` 标签的 `preload` 策略过重，列表里非当前项也会主动加载媒体。
4. 视频 URL 混合了本地文件和远程地址，远程源响应慢或不稳定，拖慢了列表和预览整体感知速度。
5. 页面状态更新会重复创建视频节点或重复请求 `/api/videos`，导致媒体元素重新初始化，体感变慢。

## Evidence

- 待收集：页面中 `<video>` 渲染方式、`preload` 配置、视频 URL 类型、网络请求与渲染行为。
