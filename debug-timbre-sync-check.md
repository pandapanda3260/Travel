[OPEN] timbre-sync-check

# 背景
- 目标：手动触发一次音色库拉取，并确认为什么当前没有把线上音色同步到本地库。
- 现象：用户反馈本地搜索搜不到部分线上音色，希望直接拉取一次并定位问题。

# 假设
1. OpenAPI 的 AK/SK 没有配置或未生效，导致运行时直接判定 `timbreApiEnabled = false`，根本没有发起拉取。
2. AK/SK 已配置，但 `VOLCENGINE_SPEECH_OPENAPI_PROJECT_NAME` 缺失或不正确，导致音色列表接口返回空或报错。
3. 签名或请求参数有误，调用 `ListBigModelTTSTimbres` 失败，但当前代码把异常吞掉了，所以页面只看到“没数据”。
4. 本地音色库存在旧缓存，且自动刷新条件没有满足，导致本次访问没有真正触发线上拉取。
5. 线上接口已返回数据，但本地合并/持久化流程有问题，导致目标音色没有写入 `data/timbre-library.json`。

# 计划
- 检查音色拉取相关本地配置是否具备发起请求的前提。
- 触发一次实际访问链路，观察运行时是否进入线上拉取。
- 根据返回结果判断是“未配置”、“请求失败”还是“写库失败”。

2. 自动刷新窗口未到，导致本地缓存没有被主动更新；删除本地缓存后应能强制触发真实拉取。
3. 接口能够成功返回，但目标音色 `zh_male_taocheng_uranus_bigtts` 本身不在当前项目可见音色范围中。
4. 接口请求成功但返回结构与当前解析逻辑不一致，导致写库后漏掉部分音色。
