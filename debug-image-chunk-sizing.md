[OPEN] image-chunk-sizing

# 问题
- 用户需要验证商品图片上传链路中的长图切块逻辑。
- 重点关注：
  - 原图大小
  - 计算出的最少分块数
  - 实际切成了几块
  - 每一块的大小

# 假设
- 假设 1：当前“最少分块数”计算正确，但实际切块因为重叠导致最终块数或块体积超出预期。
- 假设 2：当前“最少分块数”计算未充分考虑像素限制与体积限制叠加，导致起始分块数偏小。
- 假设 3：切块后每块使用原始编码直接输出时，某些分块体积仍然大于 9MB，说明切块策略仍需继续加细。
- 假设 4：在相对空白区域切块时，切点偏移导致实际块高波动，进而影响每块体积。
- 假设 5：用户当前看到的异常并非来自原图上传限制，而是来自分块后某一块仍超过上游模型单图限制。

# 计划
- 先添加运行时日志，记录原图大小、最少分块数、实际分块数与每块大小。
- 让用户复现一次上传，收集日志后再决定是否继续调整切块算法。
